# Two small fixes

## Fixes

- **`whatcable --monitor` no longer shows charger values after unplugging.** On battery, the system line was still labelling the battery's discharge numbers as charger input, which looked like stale data. It now reads "Battery ... discharge" and switches back the moment you plug in. Thanks @QuantumLibet ([#298](https://github.com/darrylmorley/whatcable/issues/298)).
- **Turning off "Show charging watts in the menu bar" now takes effect immediately.** The watts label used to stay in the menu bar until the app was restarted.
