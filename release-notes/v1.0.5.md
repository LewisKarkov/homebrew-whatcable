# A widget crash fix and a new update-notification toggle

This release stops the desktop widget crashing when you unplug a drive, and lets you silence "update available" alerts on their own. Plus six more cables in the database.

## Fixed

- **The widget no longer crashes when you eject a USB-C device.** If you kept a WhatCable widget in Notification Center and quit the app, ejecting a USB-C drive could crash the widget. The widget was leaving background hardware watchers running after it had finished reading, and a later plug or unplug event would land on memory that was already gone. It now shuts those watchers down cleanly. Thanks to @xnop for the report and the crash logs (M4 Studio).

## New

- **Separate toggle for app-update notifications.** Settings now has a "Notify on app updates" switch under "Notify on cable changes". You can turn off the "update available" alert while keeping your cable-change notifications. It is on by default, so nothing changes unless you want it to.
